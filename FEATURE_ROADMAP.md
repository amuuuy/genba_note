## 価格戦略

### プラン構成

| プラン | 価格 | 対象 |
|--------|------|------|
| **Free（無料）** | 0円 | 全ユーザー（制限付き） |
| **Pro（有料）** | 月額1,980円 / 年額19,800円 | 一人親方・小規模建設業者 |

### 無料版の制限

| 機能 | 無料 | Pro |
|------|------|-----|
| 見積・請求書の作成 | 合計5件まで（削除しても枠は戻らない） | 無制限 |
| 顧客登録 | 3件まで | 無制限 |
| 単価マスタ | 10件まで | 無制限 |
| PDF出力 | 透かし（SAMPLE）付き | 透かしなし |
| CSV出力 | 不可 | 可能 |
| 作業写真 | 1顧客あたり3枚まで | 無制限 |
| 収支管理 | 閲覧のみ（入力不可） | 全機能 |
| クラウドバックアップ | 不可 | 自動バックアップ ※Phase 3で実装予定 |

### 無料版の設計意図

- 5件の見積を作る頃には会社情報・顧客・単価マスタを入力済み → スイッチングコストが発生
- PDF透かしは「お客様に見せられない」→ 仕事で使うなら即Pro
- 収支は「見える」けど「入力できない」→ 「入力したい」欲求が生まれる

### 年額プランのメリット（運営側）

| メリット | 内容 |
|---------|------|
| 解約率が激減 | 1年後の継続率: 月額 約28% → 年額 約60% |
| 収益が読める | 年額分は確定収益。投資判断がしやすい |
| キャッシュが先に入る | 1人あたり16,830円が一括で入金 |
| LTVが2.5倍 | 月額16,830円 → 年額42,075円 |
| サンクコスト効果 | 「払ったから使おう」→ 継続 → 更新 |
| 決済失敗リスク減 | カード更新忘れによる意図しない解約を防止 |

> **算出前提**: 月額の月次解約率を約10%と想定（1年後継続率: 0.9^12 ≈ 28%、平均継続期間: 1/0.1 = 10ヶ月）。年額の年次更新率を約60%と想定（平均継続期間: 1/(1-0.6) = 2.5年）。LTVは手取り（Apple Small Business Program 15%手数料後）× 平均継続期間で算出。月額: 1,683円 × 10ヶ月 = 16,830円、年額: 16,830円 × 2.5年 = 42,075円。

## 機能詳細

### #1 入金リマインダー通知

- 支払期日の3日前、当日、3日後にプッシュ通知
- 通知タップ→該当請求書に直接遷移
- 既存のDocument.dueDateとstatusを活用

### #2 入金ダッシュボード

- 未入金（sent/issued）の請求書一覧を専用画面で表示
- 金額の大きい順/期日の近い順でソート
- 合計未入金額をサマリーカードで表示
- 既存のRevenueSummary（total/paid/unpaid）を拡張

### #3 写真マークアップ

- 作業前後写真に矢印・テキスト・囲み・フリーハンド描画を追加
- 編集後は新しい画像として保存（元写真は保持）
- 顧客への説明資料として活用
- 主要競合の請求書アプリにない独自差別化（Jimuu、ツカエル請求書との比較。2024年12月時点）

### #4 備考欄に写真添付

- 請求書の備考欄に写真を追加可能（最大5枚）
- PDFにも写真が含まれる
- 「施工報告書」を別に作る必要がなくなる
- 既存のFinancePhotoパターンを再利用（→ DocumentPhoto）
- PDF埋め込みはsealImage（base64）パターンを再利用

### #5 クラウドバックアップ

- Firebase Cloud Storage への自動バックアップ（Phase 3で構築するFirebase基盤を利用）
- 手動バックアップ/復元機能
- データ消失リスクの解消→解約抑止力が最も高い機能
- 技術構成: Firebase Cloud Storage をバックアップ先とし、端末側で暗号化してアップロード
- 暗号方式: AES-256-GCM（認証付き暗号）。IV（96bit）はファイルごとにランダム生成し、暗号文の先頭に付与
- 鍵管理: ユーザーパスフレーズからPBKDF2-HMAC-SHA256（ソルト: 128bitランダム、反復回数: 600,000回）で暗号鍵を導出。ソルトは暗号文メタデータに保存。パスフレーズはSecure Storage（iOS: Keychain / Android: Keystore）に保存。再インストール時はパスフレーズ再入力で復元
- 将来拡張: iCloud/Google Drive への保存先追加を検討（Phase 4以降、ユーザーが保存先を選択可能にする案）
- Firebase基盤が必要（#10, #13, #14と共通基盤）

### #6 現場（プロジェクト）管理

- 顧客の下に「現場」を作成
- 見積・請求書・作業写真・収支を現場単位で紐付け
- 現場ごとの進捗状態（着工前/施工中/完了）
- 現場一覧画面の追加

### #7 定期請求（リピート）

- 繰り返しスケジュール設定（毎月/隔月/四半期）
- テンプレートから自動で請求書を下書き生成
- 定期メンテナンス契約がある業者に必須

### #8 顧客別・期間別 損益レポート

- 顧客ごとの売上・経費・利益を集計
- 期間フィルタ（月次/四半期/年次）
- 「この顧客は儲かっているか」を可視化
- 既存の期間フィルタリング機能を拡張

### #9 会計ソフト連携CSV

- freee/マネーフォワードにそのまま取り込めるCSV形式で出力
- 各ソフトの仕訳インポート形式に対応
- 手動でのCSV取り込み作業を削減
- 税金関連の機能は提供しない（税理士法への配慮）

### #10 複数端末同期

- スマホ＋タブレットなど複数デバイスでデータ共有
- リアルタイム同期（Firestore）
- 主に複数端末を使う事業者向け
- Firebase基盤が必要
- 競合解決: LWW（Last Write Wins）+ updatedAtタイムスタンプ。同一フィールドの同時編集は後勝ち
- オフライン復帰時: Firestoreのオフラインキャッシュを活用し、復帰時に自動マージ。競合発生時はユーザーに通知
- 認証・権限モデル: Firebase Authenticationでユーザー認証。データ所有者（アカウントオーナー）が端末を招待する形式。Firestoreセキュリティルールで端末ごとの読み書き権限を制御

### #11 下請け・協力会社管理

- 外注先の会社情報・連絡先を管理
- 外注費の記録・支払い状況の追跡
- LINE/電話でのやり取りを集約
- 主に外注を使う業者向け

### #12 材料・原価管理

- 工事ごとの材料費・外注費を記録
- 売上に対する原価率・利益率を自動計算
- 赤字工事の早期発見
- 既存のFinanceEntry機能を拡張

### #13 顧客進捗ポータル

- 業者がアプリで「進捗を共有」→ URLを生成
- 顧客はブラウザで進捗タイムライン・作業写真を確認
- アプリ不要、ログイン不要（閲覧のみ）
- 見積の承認機能も搭載（限定的な書き込み操作。承認時はメール/SMS確認コードによる追加認証を必須化）
- Firebase Hosting + Firestore + Cloud Storageで構築
- セキュリティ:
  - 署名付きトークン（HMAC）による推測不能URL + 有効期限（デフォルト30日、業者が設定可能）
  - トークンの失効・再発行機能
  - アクセスログの記録（閲覧日時・IPアドレス）
  - レート制限（不正アクセス防止）
  - 閲覧用と承認用で別トークンを発行し、権限を分離
- ANDPADが数万円でやっていることの「個人事業主版」

### #14 AI材料リサーチ

- 材料名をテキスト入力→ EC APIで価格を一括検索
- 利用API: 楽天商品検索API（無料枠あり）、Amazon PA-API（無料枠あり、売上実績要件あり）、Yahoo!ショッピングAPI（無料枠あり）
  - ※ 各APIの利用条件・レート制限は登録時に確認が必要。Amazon PA-APIは売上実績がないと利用不可の場合がある
- AI要約（Claude Haiku / GPT-4o-mini）で相場感をまとめて表示
- 検索結果から単価マスタにワンタップ登録
- APIコスト: 約3円/回（内訳: AI要約 約2.5円/回（想定入力500トークン+出力200トークン）、EC API 無料枠内。月30回利用で約90円/ユーザー）
  - ※ 上限超過時はAI要約を省略しEC検索結果のみ表示するフォールバックを用意
- Firebase Cloud Functions経由でAPIキーを安全に管理
- 免責表示: 「参考価格です。実際の仕入れ価格は取引先にご確認ください」

## 方針

- **現場作業に特化する**（税金関連はfreee/マネーフォワードに任せる）
- **「現場→見積→請求→入金」の流れを完璧にする**
- まず1,980円の単一プランでユーザー獲得
- 年間プランでLTV 2.5倍＋収益安定化
- ユーザー増加に応じて価格改定を検討
